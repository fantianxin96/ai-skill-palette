# 灵感指令台版本记录

## v1.0.12｜2026-04-28

「想清楚」边界收敛版。产品可见版本仍显示 `v1.0`，构建版本更新为 `112`。

### 优化

- 收紧「产品结构草图」边界：只负责页面/模块清单、职责、输入处理输出、依赖关系和信息架构，不再顺手展开单条用户路径或 MVP 范围。
- 收紧「页面路径规划」边界：只针对某一个核心任务流，关注步骤、状态、反馈、异常路径和原型流转。
- 收紧「MVP 范围切分」边界：只负责从已有结构里砍范围，明确 v1 核心闭环、合并项、手动替代项、后置项和验收标准。

## v1.0.11｜2026-04-28

产品结构草图补充版。产品可见版本仍显示 `v1.0`，构建版本更新为 `111`。

### 新增

- 「想清楚」新增「产品结构草图」，用于在产品 Brief 之后继续拆解主用户路径、页面/模块、每页核心任务、输入处理输出、MVP 必须项、后置项、困惑点引导和信息架构流转。

### 说明

- 这张卡片适合放在「产品 Brief 定义」之后、「页面路径规划」之前使用，解决从产品定义到页面结构之间的断层。

## v1.0.10｜2026-04-28

产品 Brief 场景补充版。产品可见版本仍显示 `v1.0`，构建版本更新为 `110`。

### 优化

- 「想清楚」新增「产品 Brief 定义」，用于在项目开始变复杂时先确定产品定位、目标用户、核心场景、MVP 范围、不做什么、成功标准和新开 AI 对话可复用的项目说明。
- 原「产品经理需求梳理」改为「单功能需求梳理」，边界收窄到单个功能，避免和产品 brief 场景重复。

## v1.0.9｜2026-04-28

Web/App 交互检查拆分版。产品可见版本仍显示 `v1.0`，构建版本更新为 `109`。

### 新增

- 新增 Codex skill：`$i-web-interact`，专门检查 Web 页面/产品的交互状态完整度，覆盖 hover、focus、键盘、cursor、dropdown、popover、modal、toast、loading/success/error 等。
- 新增 Codex skill：`$i-app-interact`，专门检查移动端 App 的触控手感和原生交互状态，覆盖 tap/pressed、手势、底部弹层、导航转场、键盘弹起、安全区、触控热区和系统手势冲突。
- 「找问题」新增「Web 交互状态检查」和「App 触控手感检查」两张卡片。

### 说明

- 不再把 Web 端和移动端 App 的交互检查混在同一个 `i-animate` 场景里。
- `i-animate` 继续负责动效和微交互优化；新的两个 skill 负责先诊断交互状态和反馈链路是否完整。

## v1.0.8｜2026-04-28

UX 工作流场景化扩展版。产品可见版本仍显示 `v1.0`，构建版本更新为 `108`。

### 新增

- 新增 Codex skill：`$i-synthesize`，用于把用户反馈、访谈、竞品观察、聊天记录等整理成产品洞察和机会点。
- 新增 Codex skill：`$i-handoff`，用于整理开发交付 brief、组件状态清单和可验证的上线验收标准。
- 新增 Codex skill：`$i-recap`，用于 AI 协作复盘、设计思路复盘、下次开局上下文和社交媒体整理。
- 「想清楚」新增需求来源综合、页面路径规划、MVP 范围切分、从需求到上线路线。
- 「调风格」新增反 AI 味审美诊断、参考图风格提炼、普通页面变高级。
- 「适配上线」新增开发交付 Brief、验收标准清单。
- 「删减沉淀」升级为「复盘沉淀」，新增 AI 协作复盘、设计思路复盘、下次开局上下文、社交媒体整理。

### 说明

- 这次重点不是增加更多按钮，而是把常用能力拆成更容易判断使用时机的场景化 prompt。
- 新增能力覆盖审美提升、产品全流程、交付上线、个人/AI 协作复盘四个方向。

## v1.0.7｜2026-04-28

设计系统独立分类版。产品可见版本仍显示 `v1.0`，构建版本更新为 `107`。

### 优化

- 左侧新增独立分类「定系统」，不再把设计系统能力塞进普通「找问题」或「删减沉淀」场景。
- 将设计系统相关能力拆成 4 个真实使用场景：
  - 从满意页提炼规范：把第一页的字体、颜色、间距、组件沉淀为 `DESIGN.md`。
  - 新页面基准确认：开第二页前确认哪些规则必须沿用、哪些可以变化。
  - 风格漂移校准：中途检查第二页、第三页是否跑偏，并只做对齐。
  - 阶段复盘升级系统：做完一版后更新 `DESIGN.md`，补充新组件和边界状态。
- 悬浮窗高度从 `600` 调整为 `620`，让新增分类仍保持舒展，不压缩左侧导航。

## v1.0.6｜2026-04-28

设计系统与多页面一致性能力补充版。产品可见版本仍显示 `v1.0`，构建版本更新为 `106`。

### 新增

- 新增 Codex skill：`$i-system`，用于从满意页面提取颜色、字体、间距、组件、状态、移动端规则和文案语气，并沉淀为 `DESIGN.md`。
- 新增 Codex skill：`$i-consistency`，用于检查第二页、第三页是否与参考页面或 `DESIGN.md` 保持一致，重点覆盖字体、颜色、间距、组件、交互状态和文案语气。
- 灵感指令台新增「多页面一致性」卡片，放入「找问题」场景。
- 灵感指令台新增「设计系统沉淀」卡片，放入「删减沉淀」场景。

### 说明

- 这次不改变现有视觉、hover、导航和复制交互。
- 新增能力主要解决“第一页满意，后续页面字体、颜色、间距跑偏”的真实工作流问题。

## v1.0.0｜2026-04-28

正式 1.0 版本。这个版本基于前期 `v2.1 / build 210` 的视觉与交互打磨结果，作为对外分享和 Git 存档的稳定发布点。

### 定位

- 产品名：灵感指令台
- 英文名：AI Skill Palette
- 版本名：v1.0
- Git tag：v1.0.0

### 保留的核心体验

- 使用 `⌘⇧K` 唤起全局悬浮窗口。
- 按问题场景浏览 UX / 产品设计师常用 AI prompt。
- 支持全局搜索。
- 点击卡片复制 prompt，不自动发送，避免误触。
- 左侧导航固定行高，切换更顺滑。
- 视觉收敛为柔和粉紫、暖白和中性色体系。

### 回退说明

如果后续继续大改，可以通过 Git tag `v1.0.0` 回到这个正式版本。
